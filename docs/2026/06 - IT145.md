**Course Description:**
This course develops object-oriented programming skills in Java through the design and implementation of interactive business and engineering applications. The curriculum emphasizes writing code in a logical, structured, and organized manner while applying core object-oriented principles, using UML class diagrams, pseudocode, flowcharts, and professional development tools such as IDEs and SDKs.

---

## Key Topics & Concepts

This course focuses on the transition from procedural scripting to structured Object-Oriented Programming (OOP) in Java:

* **Software Design Fundamentals:** Translating client specifications and software requirements into high-level implementation plans using pseudocode and flowcharts before writing production code.
* **Object-Oriented Programming Principles:**
  * **Encapsulation:** Organizing data and behavior into classes while establishing clear boundaries that reduce unintended side effects.
  * **Abstraction:** Exposing stable interfaces while protecting implementation details, allowing internal changes without breaking external behavior.
  * **Inheritance:** Designing parent-child relationships to centralize shared logic and reduce duplication across related classes.
  * **Polymorphism:** Treating different object types through a common interface so that systems remain extensible as new implementations are introduced.
* **Class Architecture:** Interpreting UML class diagrams to implement attributes, constructors, accessors, mutators, and class hierarchies in Java.
* **Data Structures:** Using arrays and `ArrayList` collections to manage groups of related objects in menu-driven applications.
* **Program Flow & Control:** Applying conditional logic, `while` loops, `do-while` loops, and `for` / enhanced `for` loops to guide user interaction and program state.
* **Input Safety & Reliability:** Implementing input validation and exception handling to prevent crashes and ensure predictable user-facing behavior.
* **Professional Standards:** Using Java API documentation to locate existing methods, while applying naming conventions, white space, and in-line comments to improve readability and maintainability.
* **Multi-Class Program Design:** Building programs that distribute responsibilities across multiple cooperating classes rather than placing all logic in a single file.

## Major Projects & Assignments

The course used specification documents and rubric-driven development to simulate professional software engineering tasks.

1. **Project One: Global Rain Pet Boarding System**
    * **Focus:** Implementing a `Pet` class from a UML diagram and planning client-facing functionality through software design artifacts.
    * **Application:** Developed a Java class with appropriate data types, constructors, and accessor/mutator methods based on the software specification.
    * **Deliverables:** Produced a summary report containing high-level pseudocode and a flowchart for either pet check-in or pet check-out, aligned directly to the client requirements.

2. **Module One Milestone: Pseudocode and Flowchart**
    * **Focus:** Planning program logic before implementation.
    * **Application:** Broke a software requirement into ordered computational steps, including prompts, decisions, and output paths, then expressed that logic visually in a flowchart.

3. **Module Two Assignment: UML to Java Class**
    * **Focus:** Translating a UML class diagram into working Java code.
    * **Application:** Implemented either the `Dog` or `Cat` class with appropriate attributes, a constructor that initialized all values, and complete accessor/mutator coverage.

4. **Module Five Milestone: Multi-Class Java Program**
    * **Focus:** Extending an existing object-oriented codebase.
    * **Application:** Created a `Monkey` class that inherited from `RescueAnimal`, added a menu loop in `Driver.java`, created a monkey `ArrayList`, and implemented intake logic for new monkey records.

5. **Module Six Assignment: Debugging, Validation, and Exception Handling**
    * **Focus:** Improving program correctness and reliability.
    * **Application:** Debugged an existing paint calculator, corrected faulty logic, and added loops plus exception handling to validate all user input and prevent runtime failure.

6. **Module Seven Assignment: Java API Documentation**
    * **Focus:** Learning to use official documentation rather than reinventing standard functionality.
    * **Application:** Located and implemented an appropriate `Math` class method to round paint can calculations upward to the nearest whole number while satisfying all provided test cases.

7. **Project Two: Grazioso Salvare Rescue Animal System**
    * **Focus:** Completing a larger multi-class Java application with inheritance, validation, and user interaction.
    * **Application:**
        * **`Monkey.java`:** Created a subclass of `RescueAnimal` with its own attributes, constructor, and accessor/mutator methods.
        * **`Driver.java`:** Implemented a menu loop with validation, completed intake methods for animals, added reservation logic, and produced readable printed output from stored collections.
    * **Industry Standards:** Emphasized maintainable structure, in-line comments, and clear naming conventions across all modified and newly created code.

## Discussion Work & Academic Themes

1. **Introduction to the Coding Community**
    * Introduced myself as a returning student with 12 to 13 years of professional software engineering experience, especially in backend systems, database design, and performance-oriented development.
    * Framed the course as an opportunity to strengthen the theoretical foundations behind techniques I had already used professionally, especially data structures and algorithmic reasoning.
    * Highlighted the role of structured study habits, knowledge management, and disciplined scheduling in balancing coursework with full-time technical work.

2. **Object-Oriented Programming Principles**
    * Examined encapsulation, abstraction, inheritance, and polymorphism not as isolated definitions, but as a unified framework for reducing maintenance cost and controlling system complexity.
    * Connected OOP principles to professional concerns such as DRY design, dependency injection, testing, interface stability, and scalable architecture.
    * Reflected on the value of revisiting familiar engineering concepts from a more formal academic perspective.

3. **Final Discussion: Course Reflection and Multi-Class Design**
    * **Focus:** Reflecting on the most meaningful Java concepts from the course, the challenges involved in Project Two, and the architectural value of organizing the Grazioso Salvare application across separate classes.
    * **Reflection:** Emphasized how Java’s class-based object-oriented model supports maintainability through separation of concerns, and how structuring the program across `Driver`, `RescueAnimal`, `Dog`, and `Monkey` made the system easier to extend, reason about, and maintain over time.

## Personal Reflection & Takeaways

The most valuable part of this course was the opportunity to revisit object-oriented programming from a more formal and structured perspective. Although I entered the course with extensive professional experience, the class reinforced the discipline of moving from requirements to design artifacts and then from design to implementation. Writing pseudocode, interpreting UML diagrams, and building Java classes directly from software specifications emphasized that effective software engineering begins with clear structure rather than improvised coding.

One of the strongest takeaways was the role of OOP as a tool for managing complexity. Encapsulation, abstraction, inheritance, and polymorphism are often introduced as vocabulary terms, but in practice they form a framework for building systems that are easier to maintain, test, and extend. Revisiting these principles academically helped connect day-to-day professional practices such as dependency injection, modular architecture, and reusable design back to their conceptual roots.

The most challenging aspects of the course centered on working within tightly specified academic requirements while still producing clean, maintainable Java code. Projects such as the Grazioso Salvare system required careful attention to inheritance, menu-driven control flow, `ArrayList` management, and validation of user input. Assignments involving debugging and exception handling further reinforced that reliable software depends not only on correct logic, but also on how a program responds to invalid or unexpected input.

This course strengthened the bridge between practical software engineering and formal computer science study. It reinforced my belief that sound engineering is built on disciplined structure, readable code, and respect for the long-term cost of complexity. As I continue through my degree, the habits reinforced here—clear design, strong boundaries, careful validation, and deliberate implementation—align closely with the rigorous standards I hope to bring to both software engineering practice and the broader field of computer science.
