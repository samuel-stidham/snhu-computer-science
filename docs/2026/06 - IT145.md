**Course Description:**
Students use programming as a problem-solving technique in business and engineering applications, incorporating key concepts of object orientation into their programming. The course emphasizes writing code in a logical, structured, and organized manner while working with Software Development Kits (SDKs) and Integrated Development Environments (IDEs).

## Key Topics & Concepts

This course focuses on the transition from procedural scripting to robust Object-Oriented Programming (OOP) using Java:

* **Software Design Fundamentals:** Translating software specifications and client requirements into high-level solutions using pseudocode and flowcharts before programming.
* **The Four Principles of OOP:**
  * **Abstraction:** Identifying essential characteristics and identifying "what" a class does without needing to know "how".
  * **Encapsulation:** Grouping data and methods into classes to organize code into easy-to-understand parts and restricting access through modifiers.
  * **Inheritance:** Creating parent/child relationships that allow classes to share common functionality, such as a "Pet" parent class with "Dog" and "Cat" child classes.
  * **Polymorphism:** The ability for objects to be treated as instances of their parent class while maintaining unique behaviors.
* **Class Architecture:** Using Unified Modeling Language (UML) class diagrams to create graphical representations of object characteristics, methods, and class hierarchies.

* **Data Structures:** * **Arrays:** Used to store multiple values of the same type in one variable with a fixed size.
  * **ArrayLists:** Resizable lists that hold objects and provide flexibility by allowing items to be added or deleted dynamically.
* **Program Flow & Control:** Managing the order of instructions through comparative operators, `while` loops, `do-while` loops, and `for`/`for-each` loops.
* **Information Handling:** Implementing input validation and exception handling (failing gracefully) using the `Scanner` class to manage manual user input and prevent runtime crashes.
* **Professional Standards:** Utilizing Java API documentation to research methods and applying industry best practices like naming conventions and in-line comments to improve readability and maintainability.
* **Advanced OOP Structures:** Implementing **Abstract Classes** to guide subclass design and using **Interfaces** to achieve functionality similar to multiple inheritance.

## Major Projects & Assignments

The course curriculum leverages software specification documents to solve real-world problems through logical programming.

1. **Project One: Global Rain Pet Boarding System (Pet.java)**
    * **Focus:** Creating a foundational `Pet` class from a UML class diagram and applying encapsulation and inheritance principles.
    * **Application:** Developed a Java class with appropriate data structures, constructors, and accessor/mutator methods.
    * **Deliverables:** A summary report containing high-level pseudocode and a flowchart for pet check-in or check-out processes, aligned with client specification documents.

2. **Project Two: Grazioso Salvare Rescue Animal System**
    * **Focus:** Developing a complex multi-class system for an international search-and-rescue animal training company.
    * **Application:** * **Monkey.java:** Created a new class that inherits from `RescueAnimal`, implementing specific attributes, constructors, and accessors/mutators.
        * **Driver.java:** Modified the main driver to include a robust menu loop with input validation.
        * **Functional Methods:** Implemented `intakeNewMonkey()`, `reserveAnimal()`, and `printAnimals()` to manage search-and-rescue assets across multiple `ArrayLists`.
    * **Industry Standards:** Emphasized the use of in-line comments to document logic and adherence to strict variable/method naming conventions.

3. **Discussion: OOP Principles and Reflection**
    * **Focus:** Analyzing how inheritance, encapsulation, abstraction, and polymorphism work together to create reusable and maintainable code.
    * **Reflection:** Evaluating the integration of these principles and identifying areas for further practice in applying OOP concepts.

4. **Milestones:**
    * **Module One:** Designing solutions via pseudocode and flowcharts based on software specification documents.
    * **Module Five:** Practicing multi-class interactions through menu loops and object intake logic.
