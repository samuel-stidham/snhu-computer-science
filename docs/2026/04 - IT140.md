**Course Description:**
This course introduces the fundamentals of programming using Python, covering data types, variables, decision branching, loops, functions, and file handling. The curriculum emphasizes translating requirements into computational solutions and writing scripts in accordance with industry-standard best practices.

---

## Key Topics & Concepts

This course focuses on developing the logic and syntax required for professional software development:

* **Basic Language Constructs:** Defining variables and mastering primitive data types (integers, floats, strings) to manage application input and output.
* **Control Flow:** Implementing decision branching (if/elif/else) and iteration (for/while loops) to manage complex program logic.
* **Data Structures:** Utilizing lists for dynamic data storage and dictionaries for key-value mapping to organize complex datasets.
* **Modularization:** Designing functions to compartmentalize code, promote reuse, and maintain "Don't Repeat Yourself" (DRY) principles.
* **Persistence & Abstraction:** Reading from and writing to files for persistent storage and an introduction to Object-Oriented Programming (OOP) through classes and modules.
* **Professional Workflow:** Mastering the Design-Develop-Test cycle using PyCharm, pseudocode, and flowcharts.

## Major Projects & Assignments

The following projects involved the end-to-end design and development of a deep-sea survival-horror text adventure game titled "Aurelia."

1. **Project One: Deep-Sea Survival Pitch (Design Phase)**
    * **Focus:** Translating narrative requirements into structural logic and storyboarding.
    * **Application:** Developed a theme set in the Aurelia research station in the Mariana Trench. I designed a map with 10 distinct rooms and planned the computational logic using pseudocode to handle player movement and inventory management.

2. **Project Two: Aurelia Salvage (Development Phase)**
    * **Focus:** Developing a fully functional Python program using industry-relevant tools like PyCharm.
    * **Application:** Implemented the "Aurelia" survival game using a nested dictionary to link rooms, items, and directions. The engine features a robust gameplay loop with input validation, modular functions for status updates, and decision branching to handle win/loss conditions based on a required inventory count of six items.

3. **Milestone: Room Navigation Logic**
    * **Focus:** Integrating dictionaries to manage dynamically linked variables.
    * **Application:** Developed the foundational logic for moving between rooms by validating user input against available dictionary keys, ensuring the player cannot move through nonexistent exits.

## Personal Reflection & Takeaways

The most valuable takeaway from this course was the shift from simply writing syntax to implementing "Computational Thinking." Learning to break down a complex requirement—like a player moving through a multi-room environment—into discrete logical steps using pseudocode was a transformative exercise. It taught me that the majority of software engineering happens in the design phase before a single line of code is actually written.

The most challenging aspect was managing program state within the main gameplay loop, particularly ensuring that global variables like the inventory and current room updated correctly across various function calls. Debugging the logic for navigation—especially handling ambiguous directions through type-checking and qualifiers—required a disciplined approach to tracking how data flowed through the nested dictionaries.

This course solidified my interest in the architectural side of computer science. Mastering Python's data structures and modular function design provides a professional toolkit I intend to refine. I am eager to apply these scripting fundamentals to more complex systems and move toward the rigorous engineering standards exemplified by figures like Donald Knuth and Leslie Lamport.
